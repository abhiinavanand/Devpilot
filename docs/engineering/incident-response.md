# Incident Response Guide

## Severity Levels
- **SEV-1**: Full outage or data loss
- **SEV-2**: Degraded core functionality
- **SEV-3**: Non-critical issues

## Response Flow
1. Detect (monitoring + alerts)
2. Triage (assign incident commander)
3. Mitigate (rollback, feature flag, scale)
4. Communicate (status updates)
5. Postmortem

## On-call Expectations
- Acknowledge alerts within 5 minutes.
- Update stakeholders every 30 minutes.

## Postmortem Template
- Summary
- Timeline
- Root cause
- Corrective actions
