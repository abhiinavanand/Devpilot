# Contributing to DevPilot AI

Thanks for your interest in contributing! This project is structured like a production SaaS platform, so we value clear changes, good documentation, and tests when relevant.

## Workflow
1. Fork the repo and create a feature branch.
2. Keep changes scoped and documented.
3. Run tests or add new ones if behavior changes.
4. Open a pull request with context and screenshots if applicable.

## Development Standards
- Use TypeScript where possible.
- Prefer shared utilities in `libs` over duplication.
- Keep API shapes documented in `docs/api`.
- Update runbooks if behavior affects ops.

## Code Review Expectations
- Clear commit messages.
- No secrets or credentials in code.
- Update docs for user-facing changes.
